# 贡献指南

欢迎为中国海洋大学飞跃手册做贡献！这份文件将指导你如何参与项目。

## 🎯 我们需要什么样的内容

### ✅ 欢迎的内容
- **真实的个人经历** - 保研、考研、出国、就业成功案例
- **详细的过程分享** - 从准备到录取/入职的完整流程
- **实用的建议** - 避坑指南、资源推荐、学习方法
- **失败的反思** - 失败经历和从中学到的教训
- **最新的信息** - 当年的政策变化、招生信息更新
- **排查错误** - 改正现有文章的不准确之处

### ❌ 不欢迎的内容
- 广告或商业推广
- 虚假或未经证实的信息
- 过度夸大或不实的宣传
- 涉及个人隐私或敏感信息
- 歧视性或不尊重的言论

## 📝 投稿步骤

### 1. Fork 仓库
```bash
# 访问项目主页
https://github.com/OUC-Feiyue/feiyue-ouc

# 点击 Fork 按钮
```

### 2. 克隆到本地
```bash
git clone https://github.com/YOUR-USERNAME/feiyue-ouc.git
cd feiyue-ouc
```

### 3. 创建新分支
```bash
# 为你的文章创建分支
git checkout -b feature/your-article-topic

# 例如：
git checkout -b feature/保研经验-2024
```

### 4. 创建文章文件

在 `_posts/` 目录下创建新文件，格式为：`YYYY-MM-DD-title.md`

**示例**：`_posts/2024-12-15-baoyanzhu-jingyan.md`

```markdown
---
title: "你的文章标题"
date: 2024-12-15
categories: [保研]
author: 你的名字
---

## 前言
简短介绍背景...

## 主要内容
...

## 总结
...
```

### 5. 编写内容

参考下方"写作规范"部分。

### 6. 提交并推送
```bash
git add _posts/YYYY-MM-DD-title.md
git commit -m "Add: 你的文章简短描述"
git push origin feature/your-article-topic
```

### 7. 提交 Pull Request

- 进入 GitHub 项目页面
- 点击 "Pull requests" 标签
- 点击 "New pull request"
- 选择你的分支
- 填写 PR 描述（参考下方模板）
- 点击 "Create pull request"

### 8. 等待审核

审核团队会在 7 天内审核你的提交并给出反馈。

## ✍️ 写作规范

### 标题规范
- 清晰、吸引人、准确反映内容
- 避免过长（20 字以内为佳）
- 包含关键信息

✅ **好的例子**
- "保研全攻略：从实验室到梦想学校"
- "英港新申请经验：从准备到录取"

❌ **不好的例子**
- "我的故事"
- "一些关于留学的思考"

### 文章结构
推荐结构如下：

```markdown
---
title: ""
date: YYYY-MM-DD
categories: [分类]
author: 作者名
---

## 前言
- 自我介绍（学院、专业、年级）
- 最终成果
- 文章主旨

## 时间规划
- 详细的时间线和阶段任务

## 准备方法
- 关键准备工作
- 资源推荐
- 经验分享

## 常见问题
Q&A 形式回答常见问题

## 总结与建议
- 成功关键
- 给学弟学妹的建议

## 最后的话
励志和鼓励

---

**关于作者**：简短介绍
```

### 内容要点

1. **真实具体**
   - 用具体数字而不是模糊表述
   - 详细描述过程而不是结论
   - 分享真实的感受和困难

2. **逻辑清晰**
   - 按时间或主题有序组织
   - 段落分明
   - 使用标题和列表结构化内容

3. **实用有价值**
   - 包含可操作的建议
   - 推荐具体的资源
   - 分享的方法可复现

4. **礼貌尊重**
   - 不诋毁其他学校或群体
   - 避免过度推崇某一条路
   - 承认选择的多样性

### 格式规范

#### 标题
```markdown
# 一级标题（文章标题，H1）
## 二级标题（章节标题）
### 三级标题（小节标题）
```

#### 列表
```markdown
- 无序列表项
- 无序列表项

1. 有序列表项
2. 有序列表项
```

#### 强调
```markdown
**粗体** 用于强调
*斜体* 用于补充
`代码` 用于特定术语
```

#### 表格
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 行1 | 行1 | 行1 |
| 行2 | 行2 | 行2 |
```

#### 引用
```markdown
> 这是一条引用
> 可以多行
```

### 文件命名规范
- 格式：`YYYY-MM-DD-title.md`
- 日期：发布日期
- 标题：小写英文，单词用 `-` 连接
- 例：`2024-12-15-baoyanzhu-jingyan.md`

## 📋 Pull Request 模板

```markdown
## 文章信息
- 标题：
- 分类：
- 作者：

## 文章简介
简单描述文章内容和主要观点

## 检查清单
- [ ] 文章已按照写作规范编写
- [ ] 文件名格式正确（YYYY-MM-DD-title.md）
- [ ] Front matter 信息完整
- [ ] 内容真实可靠
- [ ] 没有拼写或语法错误
- [ ] 图片和链接正确可用

## 其他说明
如有其他需要说明的内容，请填写在此处
```

## 🖼️ 插入图片

如需在文章中插入图片：

1. 在 `assets/images/` 目录下创建对应分类文件夹
2. 上传图片文件
3. 在文章中引用：

```markdown
![图片描述]({{ site.url }}/assets/images/category/image-name.png)
```

## 👥 行为准则

所有参与者都应该遵守以下原则：

1. **尊重多样性**
   - 认可不同的选择和路径
   - 不贬低其他专业或行业

2. **诚实守信**
   - 分享真实经历
   - 不隐瞒失败和困难

3. **相互帮助**
   - 新投稿者受到热烈欢迎
   - 审核者提供建设性反馈

4. **包容友好**
   - 尊重所有参与者
   - 禁止任何形式的骚扰或歧视

## 🔍 审核流程

### 审核标准
- ✅ 内容真实可信
- ✅ 符合格式规范
- ✅ 无重大拼写或语法错误
- ✅ 信息准确有价值
- ✅ 不含不当内容

### 反馈周期
- 通常在 3-7 天内给出反馈
- 可能会要求修改
- 修改后重新审核

### 常见反馈
- 补充更多细节信息
- 修正时间或数据错误
- 改进措辞或结构
- 补充建议或资源

## ❓ 常见问题

### Q: 投稿有时间限制吗？
A: 没有。全年接受投稿。但越及时的信息越有价值。

### Q: 投稿需要原创吗？
A: 是的。完全原创或大幅修改后的内容。

### Q: 可以修改已发布的文章吗？
A: 可以。提交新的 Pull Request 提议修改。

### Q: 为什么我的 PR 被拒绝了？
A: 审核评论中会详细说明原因。欢迎提出异议或修改重新提交。

### Q: 我想删除我的文章怎么办？
A: 联系管理员即可。
